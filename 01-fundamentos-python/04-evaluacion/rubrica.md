## 1. Corrección técnica (40%)

| Nivel | Descripción |
| --- | --- |
| Insuficiente | El código contiene errores graves o no ejecuta. |
| Básico | Ejecuta parcialmente, con fallos de lógica o dependencias mal resueltas. |
| Adecuado | Resuelve la mayoría de requisitos con corrección funcional. |
| Excelente | Resuelve todo con precisión, robustez y claridad. |

## 2. Uso de estructuras y funciones (25%)

| Nivel | Descripción |
| --- | --- |
| Insuficiente | Uso muy limitado o incorrecto de estructuras y funciones. |
| Básico | Uso parcial, con decisiones poco claras o repetitivas. |
| Adecuado | Usa estructuras y funciones de forma correcta y razonable. |
| Excelente | Usa estructuras y funciones con criterio, modularidad y limpieza. |

## 3. NumPy y tratamiento básico de datos (20%)

| Nivel | Descripción |
| --- | --- |
| Insuficiente | No aplica NumPy o lo hace de forma incorrecta. |
| Básico | Aplica operaciones simples, pero con escasa comprensión. |
| Adecuado | Usa arrays y operaciones básicas con corrección. |
| Excelente | Aprovecha NumPy con soltura y justifica su uso. |

## 4. Notebook, presentación y buenas prácticas (15%)

| Nivel | Descripción |
| --- | --- |
| Insuficiente | Notebook desordenado, sin explicaciones ni criterio. |
| Básico | Presentación mínima, con organización mejorable. |
| Adecuado | Notebook claro, legible y con buenas prácticas básicas. |
| Excelente | Presentación muy cuidada, reproducible y bien explicada. |
