# 12-proyecto-integrado

## Propósito
Proyecto final del módulo PIA: desarrollar, en equipo, una solución progresiva basada en datos, integración de varias fuentes e IA/ML aplicada a un problema elegido libremente.

El proyecto integrador de UD7 puede reutilizarse como ensayo o avance técnico, pero este proyecto final de módulo mantiene requisitos propios: tema validado, al menos dos fuentes de datos, integración real y checkpoints progresivos.

## Materiales incluidos
- **01-teoria/proyecto.md**: Guía del proyecto final, requisitos, fases de seguimiento, ejemplos y criterios generales.
- **01-teoria/mapa_mini_proyectos.md**: Relación entre mini-proyectos comunes, checkpoints y aplicación al proyecto final.
- **01-teoria/checkpoints_proyecto_final.md**: Entregas parciales, criterios de aceptación y revisión docente del proyecto final.
- **01-teoria/banco_ejemplos_proyectos.md**: Familias de proyectos, ejemplos válidos, propuestas insuficientes y mejoras posibles.
- **01-teoria/evaluacion.md**: Criterios de evaluación, si se mantiene como documento separado.
- **01-teoria/faq.md**: Preguntas frecuentes, si se mantiene como documento separado.
- **90-archivo/referencias-historicas**: Proyecto_PIA-SBD.md y .pdf del curso 24-25 (como referencia histórica)

## Prácticas asociadas
- Mini-proyectos guiados comunes durante el curso.
- Proyecto final progresivo elegido por cada equipo.

## Requisitos clave del proyecto final

- Tema libre, validado al inicio.
- Al menos dos fuentes de datos diferenciadas.
- Integración real entre fuentes, no una unión superficial.
- Limpieza y curación de datos.
- Componente de IA/ML justificado.
- Salida útil: API, dashboard, informe, asistente, recomendador, predicción u otra interfaz razonada.
- Documentación y defensa final.
