# Preguntas frecuentes

## ¿Hace falta usar todas las herramientas de la unidad?

No. Cada grupo debe elegir solo las que tengan sentido para su proyecto y justificar por qué.

## ¿Es obligatorio usar RAG?

No. Solo tiene sentido si el proyecto necesita recuperar información relevante desde documentos o una base de conocimiento.

## ¿Hay que usar modelos abiertos?

No necesariamente. Puede usarse una API externa si encaja mejor por tiempo, recursos o facilidad de integración.

## ¿Qué pasa si una categoría no aplica al proyecto?

Debe explicarse con claridad. También es una decisión técnica válida.

## ¿Se valora más la complejidad o la coherencia?

Se valora más la coherencia. Un stack pequeño y bien justificado suele ser mejor que una arquitectura ambiciosa mal integrada.

## ¿La evaluación exige despliegue completo?

No siempre. Depende del alcance del proyecto, pero en general se prioriza que exista una versión funcional y defendible.
