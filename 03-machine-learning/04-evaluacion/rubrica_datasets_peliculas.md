# Rúbrica de Evaluación – Proyecto de Dataset Personalizado

## Objetivo
Evaluar la capacidad del alumno para diseñar, obtener, procesar y documentar un dataset original mediante fuentes abiertas (API, CSV, etc.), integrando prácticas de ingeniería de datos y herramientas profesionales como Poetry.

---

## 🧱 Criterios de Evaluación

| Criterio                            | Descripción                                                                 | Peso (%) |
|------------------------------------|-----------------------------------------------------------------------------|----------|
| **1. Elección de la fuente de datos**   | Fuente interesante, relevante, abierta y técnicamente accesible            | 10%      |
| **2. Automatización de la recogida**   | Uso correcto de API, descarga o lectura programática                       | 20%      |
| **3. Limpieza y transformación**       | Conversión a estructura útil, normalización, tratamiento de nulos          | 20%      |
| **4. Generación del dataset final**    | Guardado correcto (CSV, Parquet), estructura clara y reusable              | 10%      |
| **5. Código estructurado con Poetry**  | Proyecto ejecutable, con dependencias declaradas y separadas en módulos    | 10%      |
| **6. Documentación del proceso**       | README con explicación clara de fuente, pasos, dificultades y mejoras      | 10%      |
| **7. Calidad y relevancia del dataset**| Dataset tiene potencial para análisis reales o integrarse en flujo mayor   | 10%      |
| **8. Creatividad y profundidad**       | Nivel de personalización, esfuerzo adicional (paginación, fusión, etc.)    | 10%      |

---

## ✅ Recomendaciones para los estudiantes

- Elige una fuente real y bien documentada (OpenAQ, REE, Open-Meteo…)
- Automatiza todo el proceso para que pueda rehacerse fácilmente
- Prepara el proyecto como si fuera a integrarse en un flujo de producción
- Usa `pyproject.toml` para declarar dependencias y mantener tu entorno reproducible
- Añade comentarios y ejemplos de uso en el README
